# AI Agent Guidelines

เอกสารนี้เป็นกติกาการทำงานสำหรับ AI coding agent ที่เข้ามาช่วยพัฒนาโปรเจกต์ศรีคเนศ เทวาลัย

## User Context

- ผู้ใช้ไม่ใช่โปรแกรมเมอร์ ต้องอธิบายและพาไปทีละขั้นตอน
- ก่อนเริ่มงาน ให้อธิบายว่าจะทำอะไร ทำไปเพื่ออะไร และคาดว่าจะได้ผลลัพธ์แบบไหน
- ใช้ภาษาง่าย หลีกเลี่ยงศัพท์เทคนิคที่ไม่จำเป็น และค่อย ๆ อธิบายแนวคิดทางเทคนิคเมื่อเกี่ยวข้อง
- หลังทำงานแต่ละส่วนเสร็จ ต้องบอกขั้นตอนถัดไปที่แนะนำ เพื่อให้โปรเจกต์เดินหน้าจนครบ

## Response Language

- ตอบด้วยภาษาเดียวกับภาษาที่ผู้ใช้ใช้
- หากผู้ใช้พิมพ์ภาษาไทย ให้ตอบภาษาไทย
- หากผู้ใช้พิมพ์ภาษาอังกฤษ ให้ตอบภาษาอังกฤษ

## Working Method

1. อ่านเอกสารและโครงสร้างที่เกี่ยวข้องก่อนแก้ไข
2. อธิบายแผนสั้น ๆ ก่อนลงมือ
3. แก้เฉพาะไฟล์ที่เกี่ยวข้องกับคำขอ
4. ตรวจสอบผลลัพธ์หลังแก้ไข
5. สรุปสิ่งที่ทำ ไฟล์ที่เปลี่ยน การตรวจสอบ และขั้นตอนถัดไป

## Coding Guidelines

- อย่าใส่ Tailwind utility class หลายคลาสตรง ๆ ใน template ของ component
- หาก element ต้องใช้ Tailwind มากกว่า 1 คลาส ให้สร้าง custom class แล้วจัดการผ่าน `@apply`
- ใช้ dependencies ของโปรเจกต์ให้น้อยที่สุด
- เพิ่ม dependency เฉพาะเมื่อจำเป็นจริง และอธิบายเหตุผลก่อนเพิ่ม
- รักษา TypeScript แบบ strict และลบ code/import ที่ไม่ได้ใช้งาน
- รักษา responsive, accessibility และ design system ที่ระบุไว้ใน `DESIGN.md`

## Project Boundaries

- ห้ามเปลี่ยน architecture ใหญ่โดยไม่จำเป็น
- ห้ามลบ feature เดิมโดยไม่ได้รับอนุญาต
- ห้ามแก้ UI/UX ที่อยู่นอกเหนือคำขอ
- ห้ามแก้ `.env` หรือ secret
- ห้ามเก็บ LINE token หรือข้อมูลส่วนตัวเกินจำเป็น
- หากพบปัญหาที่ไม่เกี่ยวข้องกับงาน ให้รายงานไว้ แต่ยังไม่ต้องแก้

## Git Rules

- ห้าม commit หรือ push เอง
- หากจำเป็นต้อง commit ต้องขออนุญาตผู้ใช้ก่อน
- commit message ต้องเขียนเป็นภาษาไทย
- ห้ามใช้คำสั่งที่ทำลายข้อมูลหรือย้อนการแก้ไขของผู้ใช้โดยไม่ได้รับอนุญาต

## Verification

ก่อนสรุปงาน ให้ตรวจตามความเหมาะสม โดยงานโค้ดต้องพยายามรัน:

- `npm run lint`
- `npx tsc -b`
- `npm run build`
- ทดสอบ flow หลักที่เกี่ยวข้อง

หากตรวจสอบรายการใดไม่ได้ ต้องบอกผู้ใช้ตามจริง

## Completion Format

ใช้รูปแบบนี้เมื่อทำงานเสร็จ:

### Done

สรุปสิ่งที่ทำ

### Changed Files

- ระบุไฟล์และสิ่งที่เปลี่ยน

### Verified

- lint: pass/fail/not run
- typecheck: pass/fail/not run
- build: pass/fail/not run

### Notes

สิ่งที่ควรรู้ต่อและขั้นตอนถัดไปที่แนะนำ
