# Product

## Register

product

## Platform

web

เป็นเว็บ responsive ที่ออกแบบ mobile-first และพร้อมต่อยอดเป็น LINE LIFF โดยผู้ใช้สามารถใช้งานแบบ Guest ผ่าน browser ได้ทันที

## Users

ผู้ที่ต้องการสัมผัสประสบการณ์ขอพรและเสี่ยงโชคกับศรีคเนศ เทวาลัย ผ่านบรรยากาศที่สมจริงและใช้งานง่าย ผู้ใช้หลักไม่ควรถูกบังคับให้สมัครสมาชิกก่อนเริ่มพิธี

## Product Purpose

เปลี่ยนขั้นตอนการขอพรและจุดธูปให้เป็นพิธีดิจิทัลแบบ immersive: เข้าสู่เทวาลัย เลือกกิจกรรม เลือกองค์เทพ จุดธูป รอการเผยเลขมงคล และรับผลลัพธ์ในจังหวะที่รู้สึกมีความหมาย

ความสำเร็จของผลิตภัณฑ์วัดจาก:

- ผู้ใช้เข้าใจลำดับพิธีได้โดยไม่ต้องมีคู่มือยาว
- ผู้ใช้ทำ flow หลักจนถึงผลลัพธ์ได้สำเร็จใน browser
- ภาพ องค์เทพ และจังหวะธูปยังเป็นจุดสนใจหลักตลอดประสบการณ์
- ผู้ใช้เลือกใช้ Guest หรือรับผลผ่าน LINE ได้โดยไม่ติดข้อจำกัดด้านบัญชี

## Positioning

ประสบการณ์เทวาลัยดิจิทัลที่ให้ความรู้สึกสงบ ลึก และหรูหรา มากกว่าเว็บเกมทั่วไปหรือหน้าขายสินค้า โดยใช้ภาพสถานที่จริงเป็นแกนของประสบการณ์

## Brand Personality

หรูหรา

## Anti-references

- เว็บ AI template สำเร็จรูป
- เว็บเกมเสี่ยงโชคทั่วไป
- เว็บ landing page ขายสินค้า
- ภาพเทพหรือวัตถุที่ดูปลอม/ผิดรูป

## Design Principles

- Image first: ภาพเทวาลัยและองค์เทพต้องเป็นพระเอกของทุกหน้าจอ
- Sacred pacing: ให้จังหวะการเลือก จุดธูป และเผยผลลัพธ์มีความตั้งใจ ไม่เร่งหรือกระพริบเกินจำเป็น
- Quiet luxury: ใช้ทอง แดงไวน์ ดำ และแสงอุ่นอย่างมีลำดับ ไม่ใช้เอฟเฟกต์จนกลบภาพ
- Direct interaction: ปุ่มและข้อความต้องชัด แต่วางเป็น overlay เหนือภาพโดยไม่สร้างแผงเบลอด้านล่าง
- Trustworthy reveal: เลขต้องสุ่มจากระบบเมื่อพิธีจบ และไม่ทำให้ผู้ใช้เข้าใจว่าเป็นผลการเสี่ยงโชคที่รับประกันได้

## Accessibility & Inclusion

- รองรับการใช้งานแบบ Guest โดยไม่ต้องส่งข้อมูล LINE ใน browser และไม่เก็บข้อมูล LINE ใน local storage
- ปุ่มและพื้นที่กดมีขนาดอย่างน้อย 44 x 44 px พร้อม focus state ที่มองเห็นได้
- ข้อความต้องมี contrast เพียงพอบนภาพ และมี text shadow เมื่อวางบนฉากสว่าง
- รองรับ `prefers-reduced-motion` โดยลด particle, smoke loop และเปลี่ยน transition เป็น fade สั้น
- เสียงเป็น optional และต้องมีปุ่มปิดเสียงที่เข้าถึงได้ด้วยคีย์บอร์ด
- การเชื่อมต่อ LINE ต้องเป็นทางเลือก และยังคงแสดงผลลัพธ์ให้ Guest ได้เสมอ
