# UX Flow

เอกสารนี้เป็น state-level contract สำหรับ AI coding agent และ QA ของ flow หลัก

## Flow

```text
Welcome / Intro
  -> เริ่มการขอโชค
เลือกองค์เทพ
  -> swipe carousel -> เลือกองค์
เสี่ยงโชค
  -> จุดธูปขอโชค -> เห็นเลขชัด 1 ตัว + เลขจาง 1 ตัว + ?
  -> เลือกสนับสนุนผ่าน Beam หรือรับเลข 2 หลัก
  -> ชำระสำเร็จ: เปิดเลข 3 ตัว -> เล่นเอฟเฟกต์อีกครั้ง -> เปิดเลข 2 ตัวอีกชุด
Result
  -> แสดงเลขแนวนอน: 3 ตัวแถวบน และ 2 ตัวแถวล่าง
  -> ผู้สนับสนุนบันทึกการ์ดเลขมงคล | รับผลผ่าน LINE (เฉพาะ LIFF) | เริ่มใหม่
```

## States

| State | User sees | Primary action |
| --- | --- | --- |
| `welcome` | ฉากซุ้มเทวาลัย/องค์ประธาน | `เริ่มการขอโชค` |
| `activity` | ฉากทางเดินพรมแดง (ซ่อนไว้ชั่วคราว) | ยังไม่เปิดจาก flow ปัจจุบัน |
| `deity` | 3D carousel ขององค์เทพ | เลือกองค์และไปต่อ |
| `incense-idle` | ธูปหนึ่งดอก เลข `?` แนวตั้ง | `จุดธูปขอโชค` |
| `incense-burning` | ไฟ ควัน progress, เลขแรกชัด, เลขที่สองจาง 15% และ `?` | รอให้พิธีจบ; ปุ่มไม่เริ่มซ้ำ |
| Modal | ตัวเลือกสนับสนุนสีเขียวหรือตัวเลือกไม่สนับสนุนสีแดง | สแกน QR หรือยืนยันรับเลข 2 หลัก |
| `result` | ผู้สนับสนุนเห็นเลข 3 + 2 หลักสองแถว; ผู้ไม่สนับสนุนเห็นเลข 2 หลัก | บันทึกการ์ด, LINE เฉพาะ LIFF, เริ่มใหม่ |
| `wish-placeholder` | สถานะสำรองของ branch ขอพรที่ซ่อนไว้ชั่วคราว | ยังไม่เปิดจากหน้าใช้งาน |

## Guest and LINE

- Browser เริ่มเป็น `Guest` และสร้าง Firebase Anonymous ID แบบเงียบ ๆ โดยไม่แสดงหน้า login
- Anonymous ID คงอยู่ใน browser เดิมและใช้กู้ `drawId` ล่าสุด รวมถึงควบคุมโควต้าฟรี
- Guest และ LINE สามารถสร้าง QR สนับสนุนได้เหมือนกัน
- เมื่อ QR ยังไม่ถูกชำระ ผู้ใช้ยังสลับไปเลือกรับเลขฟรีได้ และเมื่อสลับกลับจะใช้ QR/Charge เดิมที่ยังไม่หมดอายุ
- ผู้ใช้ที่กลับเข้าแอปจาก browser หรือบัญชี LINE เดิมจะถูกพาไปยัง `drawId` ล่าสุดที่ยังค้าง และใช้เลขเดิม
- รับเลขฟรีได้สูงสุด 2 รอบต่อวันต่อ Anonymous ID หรือ LINE User ID; รอบสนับสนุนไม่ถูกจำกัดด้วยโควต้านี้
- เมื่อ Beam ยืนยันสำเร็จ ระบบเปิดผลอัตโนมัติโดยไม่ให้ผู้ใช้เลือก Radio ซ้ำ
- Browser mode ห้ามเขียน LINE profile หรือ LINE token ลง local storage
- LIFF mode ใช้เฉพาะเมื่อ runtime ตรวจพบและ initialize สำเร็จ
- ปุ่มรับผลผ่าน LINE แสดงเฉพาะ LIFF; ผู้สนับสนุนจะได้รับการ์ดเลขมงคลอัตโนมัติเมื่อ LIFF อนุญาตให้ส่งข้อความ
- การ์ดผู้สนับสนุนใช้หัวข้อ `เลขเสี่ยงโชคมงคล` และ panel แบบ forest glass สีม่วงไล่เขียวโปร่ง 50%
- `line-pending`: แสดง loading สั้น ๆ และกันการกดซ้ำ
- `line-success`: แจ้งว่า `ส่งผลผ่าน LINE แล้ว` และคง result ไว้
- `line-error`: แจ้ง `ยังเชื่อมต่อ LINE ไม่สำเร็จ` พร้อม `ลองใหม่` และ `ใช้งานต่อแบบ Guest`

## Loading and errors

- เปิดพิธี: `กำลังเปิดประตูเทวาลัย...`
- จุดธูป: `กำลังอธิษฐาน...`
- เผยเลข: `กำลังเปิดเผยเลขมงคล...` เป็นเวลา 3 วินาที
- โหลดภาพไม่สำเร็จ: คง background สีดำและ warm glow แล้วแสดงข้อความ/ปุ่มต่อ
- สุ่มผลลัพธ์ผิดพลาด: กลับ `incense-idle` พร้อม toast `ลองเริ่มพิธีอีกครั้ง`

## UX invariants

- ในเวอร์ชันปัจจุบันข้ามหน้าเลือกกิจกรรมและตั้งค่าเป็น `เสี่ยงโชค` อัตโนมัติ
- หน้าเลือกกิจกรรมและ branch `ขอพร` เก็บไว้พัฒนาภายหลัง
- เลขธูประหว่างพิธีต้องเรียงแนวตั้ง
- เลขสุ่มต้องสร้างเมื่อเริ่มพิธีและเปิดเผยผลเมื่อ ritual complete ไม่ hardcode จาก mockup
- ทุกหน้าต้องมี back behavior ที่ไม่ทำให้ข้อมูลที่เลือกเสียโดยไม่จำเป็น
